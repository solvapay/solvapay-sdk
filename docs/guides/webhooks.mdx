## Overview

SolvaPay sends webhooks to notify your application about important events:

- **Purchase Created** - New purchase activated
- **Purchase Updated** - Purchase plan or status changed
- **Purchase Cancelled** - Purchase cancelled
- **Payment Succeeded** - Payment processed successfully
- **Payment Failed** - Payment processing failed
- **Transaction Events** - Payment, fee, refund, commission, transfer
- **Customer / Product / Plan Events** - Resource lifecycle events

### Webhook Flow

```
SolvaPay Backend
    ↓
Enqueue WebhookEvent + WebhookDelivery
    ↓  (cron every 30 s, up to 20 per batch)
POST to Your Endpoint
    ↓
Verify SV-Signature
    ↓
Process Event
    ↓
Return 2xx
```

## Webhook Payload

Every webhook delivery is a JSON POST with this structure:

```json
{
  "id": "evt_abc123def456",
  "type": "purchase.created",
  "created": 1740000000,
  "api_version": "2025-10-01",
  "data": {
    "object": { "..." },
    "previous_attributes": null
  },
  "livemode": false,
  "request": {
    "id": null,
    "idempotency_key": null
  }
}
```

### Headers

| Header | Description |
|---|---|
| `SV-Signature` | `t={unix_ts},v1={hmac_hex}` — used for verification |
| `SV-Event-Id` | Unique event ID (`evt_…`) |
| `SV-Delivery` | Unique delivery ID (`dlv_…`) — use for idempotency |
| `User-Agent` | `SolvaPay/1.0 (+webhooks)` |
| `Content-Type` | `application/json` |

### Signature Scheme

The HMAC-SHA256 is computed over `"{timestamp}.{rawBody}"` using the full webhook
signing secret (including the `whsec_` prefix) as the key. The result is sent in
the `SV-Signature` header as `t={timestamp},v1={hex_digest}`.

## Webhook Setup

### 1. Configure Webhook Endpoint

Create an endpoint in the SolvaPay dashboard or via the API:

```
https://your-app.com/api/webhooks/solvapay
```

### 2. Create Webhook Endpoint

Create an endpoint to receive webhooks:

```typescript
// app/api/webhooks/solvapay/route.ts (Next.js App Router)
import { NextRequest, NextResponse } from 'next/server'
import { verifyWebhook } from '@solvapay/server'

export async function POST(request: NextRequest) {
  try {
    const body = await request.text()
    const signature = request.headers.get('sv-signature')

    if (!signature) {
      return NextResponse.json({ error: 'Missing signature' }, { status: 401 })
    }

    const event = verifyWebhook({
      body,
      signature,
      secret: process.env.SOLVAPAY_WEBHOOK_SECRET!,
    })

    await handleWebhookEvent(event)

    return NextResponse.json({ received: true })
  } catch (error) {
    console.error('Webhook error:', error)
    return NextResponse.json({ error: 'Webhook processing failed' }, { status: 500 })
  }
}
```

### Express.js Webhook Endpoint

```typescript
// routes/webhooks.ts
import express from 'express'
import { verifyWebhook } from '@solvapay/server'

const router = express.Router()

// Use express.raw() so the body is available as a Buffer for signature verification
router.post('/solvapay', express.raw({ type: 'application/json' }), async (req, res) => {
  try {
    const signature = req.headers['sv-signature'] as string

    if (!signature) {
      return res.status(401).json({ error: 'Missing signature' })
    }

    const event = verifyWebhook({
      body: req.body.toString(),
      signature,
      secret: process.env.SOLVAPAY_WEBHOOK_SECRET!,
    })

    await handleWebhookEvent(event)

    res.json({ received: true })
  } catch (error) {
    console.error('Webhook error:', error)
    res.status(500).json({ error: 'Webhook processing failed' })
  }
})

export default router
```

## Webhook Verification

### Verify Webhook Signature

Always verify webhook signatures to ensure requests are from SolvaPay:

```typescript
import { verifyWebhook } from '@solvapay/server'

// verifyWebhook parses the SV-Signature header (t=…,v1=…), recomputes
// the HMAC, and throws if the signature is invalid or too old (> 300 s).
const event = verifyWebhook({
  body: rawBodyString,
  signature: svSignatureHeader,
  secret: process.env.SOLVAPAY_WEBHOOK_SECRET!,
})
```

### Edge Runtimes

For Vercel Edge Functions, Cloudflare Workers, and Deno, import from the edge entry point.
The edge version uses Web Crypto and returns a `Promise`:

```typescript
import { verifyWebhook } from '@solvapay/server/edge'

const event = await verifyWebhook({
  body: rawBodyString,
  signature: svSignatureHeader,
  secret: process.env.SOLVAPAY_WEBHOOK_SECRET!,
})
```

### Environment Variables

Set your webhook secret:

```env
SOLVAPAY_WEBHOOK_SECRET=whsec_...
```

Get your webhook secret from the SolvaPay dashboard (shown once at endpoint creation, or via the reveal endpoint).

## Event Handling

### Event Types

Handle different event types:

```typescript
interface WebhookEvent {
  id: string
  type: string
  created: number
  api_version: string
  data: {
    object: Record<string, any>
    previous_attributes?: Record<string, any> | null
  }
  livemode: boolean
  request: {
    id?: string | null
    idempotency_key?: string | null
  }
}
```

### Handle Events

```typescript
async function handleWebhookEvent(event: WebhookEvent) {
  switch (event.type) {
    case 'purchase.created':
      await handlePurchaseCreated(event.data.object)
      break

    case 'purchase.updated':
      await handlePurchaseUpdated(event.data.object)
      break

    case 'purchase.cancelled':
      await handlePurchaseCancelled(event.data.object)
      break

    case 'payment.succeeded':
      await handlePaymentSucceeded(event.data.object)
      break

    case 'payment.failed':
      await handlePaymentFailed(event.data.object)
      break

    default:
      console.log('Unhandled event type:', event.type)
  }
}
```

### Purchase Created

```typescript
async function handlePurchaseCreated(object: any) {
  const { id, customer, plan, product, amount, currency } = object

  await db.purchases.create({
    purchaseId: id,
    customerRef: customer,
    planRef: plan,
    productRef: product,
    amount,
    currency,
    status: 'active',
  })

  await clearPurchaseCache(customer)
}
```

### Purchase Cancelled

```typescript
async function handlePurchaseCancelled(object: any) {
  const { id, customer } = object

  await db.purchases.update({
    where: { purchaseId: id },
    data: { status: 'cancelled', cancelledAt: new Date() },
  })

  await clearPurchaseCache(customer)
}
```

### Payment Succeeded

```typescript
async function handlePaymentSucceeded(object: any) {
  const { id, amount, currency, customer } = object

  await db.payments.create({
    paymentIntentId: id,
    customer,
    amount,
    currency,
    status: 'succeeded',
  })

  await clearPurchaseCache(customer)
}
```

## Complete Examples

### Next.js Complete Example

```typescript
// app/api/webhooks/solvapay/route.ts
import { NextRequest, NextResponse } from 'next/server'
import { verifyWebhook } from '@solvapay/server'

export async function POST(request: NextRequest) {
  try {
    const body = await request.text()
    const signature = request.headers.get('sv-signature')

    if (!signature) {
      return NextResponse.json({ error: 'Missing signature' }, { status: 401 })
    }

    const event = verifyWebhook({
      body,
      signature,
      secret: process.env.SOLVAPAY_WEBHOOK_SECRET!,
    })

    await handleWebhookEvent(event)

    return NextResponse.json({ received: true })
  } catch (error) {
    console.error('Webhook error:', error)
    return NextResponse.json({ error: 'Webhook processing failed' }, { status: 500 })
  }
}

async function handleWebhookEvent(event: any) {
  const { type, data } = event

  switch (type) {
    case 'purchase.created':
    case 'purchase.updated':
    case 'purchase.cancelled':
      if (data.object.customer) {
        await clearPurchaseCache(data.object.customer)
      }
      await updatePurchaseInDatabase(data.object)
      break

    case 'payment.succeeded':
    case 'payment.failed':
      await handlePaymentEvent(type, data.object)
      break

    default:
      console.log('Unhandled event type:', type)
  }
}
```

## Best Practices

1. **Always Verify Signatures** — Never process webhooks without verifying `SV-Signature`.

2. **Use the Delivery ID for Idempotency** — The `SV-Delivery` header is unique per delivery attempt. Store processed delivery IDs and ignore duplicates.

3. **Respond Quickly** — Return a 2xx within 10 seconds. Move heavy processing to a background queue.

4. **Handle Unknown Events Gracefully** — Log and return 200 for unrecognised `type` values so the delivery succeeds.

5. **Expect Retries** — Deliveries are retried with exponential backoff (1 min → 72 hours, up to 12 attempts). After 12 failures the endpoint is auto-disabled.

6. **Keep the Raw Body** — Signature verification requires the exact body bytes. Use `express.raw()` or `request.text()`, not a parsed JSON object.

Use the dashboard test button or the API:

```bash
curl -X POST https://api.solvapay.com/v1/ui/webhooks/{endpointId}/test \
  -H "Authorization: Bearer $TOKEN"
```

This sends a `payment.succeeded` test event to the endpoint.

## Retry Schedule

| Attempt | Delay |
|---|---|
| 1 | 60 s |
| 2 | 5 min |
| 3 | 15 min |
| 4 | 1 hour |
| 5 | 6 hours |
| 6 | 24 hours |
| 7 | 48 hours |
| 8+ | 72 hours |

Max 12 attempts. After that the endpoint is disabled automatically.
