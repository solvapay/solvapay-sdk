# @solvapay/mcp-express changelog

From `0.1.1` onwards this changelog is generated by
[changesets](https://github.com/changesets/changesets) — entries below
the inaugural release are maintained by hand.

## 0.1.0

Inaugural release. Node `(req, res, next)` OAuth bridge middleware for
the SolvaPay MCP server:

- `createMcpOAuthBridge(options)` — the turnkey Express middleware
  that mounts `/.well-known/oauth-protected-resource`,
  `/.well-known/oauth-authorization-server`, and the
  `/oauth/{register,authorize,token,revoke}` relay endpoints in one
  shot, proxying client-registration, authorization, token-exchange,
  and token-revocation requests to the SolvaPay upstream.
- Individual handler factories —
  `createOAuthProtectedResourceHandler`,
  `createOAuthAuthorizationServerHandler`,
  `createOAuthRegisterHandler`,
  `createOAuthAuthorizeHandler`,
  `createOAuthTokenHandler`,
  `createOAuthRevokeHandler` — for integrators who want to own their
  router and cherry-pick pieces of the bridge.
- Byte-for-byte body passthrough on `/oauth/token` and
  `/oauth/revoke` (handles `+` vs `%20` encoding quirks in
  `application/x-www-form-urlencoded` bodies correctly).
- Native-scheme CORS helpers that mirror `Access-Control-Allow-Origin`
  for `cursor://`, `vscode://`, `vscode-webview://`, and `claude://`
  clients.

Peers on `@solvapay/mcp-core` for the framework-neutral discovery JSON
builders, bearer helpers, and OAuth contract types.

### Extraction history

The Node OAuth middleware lived on the old `@solvapay/mcp` package
before the reshuffle. It was extracted verbatim — no behavioural
changes — so that `@solvapay/mcp-core` could be strictly
framework-neutral (no `req` / `res` / `next`), and so that the Node
middleware could evolve independently of the fetch-first bridge in
`@solvapay/mcp-fetch`.

Consumers upgrading from the old `@solvapay/mcp` path:

```diff
- import { createMcpOAuthBridge } from '@solvapay/mcp'
+ import { createMcpOAuthBridge } from '@solvapay/mcp-express'
```

No option shape changes.
